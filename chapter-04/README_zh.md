## 习题

### 习题 1

**考虑以下 CUDA 内核及其对应的调用它的主机函数：**
```cpp
01 __global__ void foo_kernel(int* a, int* b) {
02     unsigned int i = blockIdx.x*blockDim.x + threadIdx.x;
03     if(threadIdx.x < 40 || threadIdx.x >= 104) {
04         b[i] = a[i] + 1;
05     }
06     if(i%2 == 0) {
07         a[i] = b[i]*2;
08     }
09     for(unsigned int j = 0; j < 5 - (i%3); ++j) {
10         b[i] += j;
11     }
12 }
13 void foo(int* a_d, int* b_d) {
14     unsigned int N = 1024;
15     foo_kernel <<< (N + 128 - 1)/128, 128 >>>(a_d, b_d);
16 }
```

**a. 每个块有多少个 warp？**

每个 warp 包含 32 个线程；每个块有 128 个线程（`<<<...>>>` 中的第二个参数），因此每个块有 `128/32=4` 个 warp。

**b. 网格中共有多少个 warp？**

共有 `(N + 128 - 1)/128 = (1024+128-1)/128 = 8` 个块，每个块有 4 个 warp（见 a）；因此网格中共有 32 个 warp。

**c. 对于第 04 行的语句：**

**i. 网格中有多少个 warp 是活跃的？**

第一个 warp（warp 0）执行线程 0 到 31；所有线程都满足条件（`threadIdx.x < 40`），因此该 warp 是活跃的。

第二个 warp（warp 1）执行线程 32–63。由于部分线程满足条件（`threadIdx.x < 40`），根据 CUDA 的 SIMT 模型，warp 中所有线程都会运行，只是部分线程处于非活跃状态。

第三个 warp（warp 2）执行线程 64–95，没有线程满足 if 条件（`threadIdx.x < 40 || threadIdx.x >= 104`），所有线程被跳过，该 warp 不活跃。

第四个 warp（warp 3）执行线程 96 到 127，与第二个 warp 类似，部分线程（`threadIdx.x >= 104`）是活跃的，因此该 warp 会运行。

所以每个块有 3 个活跃 warp，网格中共有 `3×8=24` 个活跃 warp。

**ii. 网格中有多少个 warp 存在分支分歧？**

如上所述，warp 1 和 warp 3 存在分歧（warp 中只有部分线程活跃），因此每个块有 2 个分歧 warp，网格中共有 `8×2=16` 个分歧 warp。

**iii. 块 0 的 warp 0 的 SIMD 效率是多少（%）？**

warp 中有 32 个线程（0 到 31），全部在执行，因此效率为 `32/32 = 100%`。

**iv. 块 0 的 warp 1 的 SIMD 效率是多少（%）？**

Warp 1 覆盖线程 `[32,63]`。线程 `32-39` 在执行，而线程 `40-63` 不执行（`threadIdx.x < 40`）。因此 SIMD 效率为 `8/32=0.25=25%`。

**v. 块 0 的 warp 3 的 SIMD 效率是多少（%）？**

Warp 3 覆盖线程 96–127。线程 `96-103` 不活跃，而线程 104-127 在执行。因此 SIMD 效率为 `24/32=0.75=75%`。

**d. 对于第 07 行的语句：**

  **i. 网格中有多少个 warp 是活跃的？**

  网格中每隔一个线程就是活跃的，意味着每个 warp 中有一半线程是活跃的，因此所有 32 个 warp 都是活跃的。

  **ii. 网格中有多少个 warp 存在分支分歧？**

  所有 warp 都存在分歧，因此共有 32 个分歧 warp。

  **iii. 块 0 的 warp 0 的 SIMD 效率是多少（%）？**

  warp 中有一半线程是活跃的，因此 SIMD 效率为 `16/32=0.5=50%`。

**e. 对于第 09 行的循环：**

`i` 的范围从 0 到 1023（`8 × 128=1024` 个线程）。`i%3` 可以取 3 个值 `{0, 1, 2}`：342 个零、341 个一、341 个二。

  ```cu
09     for(unsigned int j = 0; j < 5 - (i%3); ++j)
10         b[i] += j
11     }
  ```

因此第 10 行在 342 种情况下执行 5 次（5-0），在 341 种情况下执行 4 次（5-1），在 341 种情况下执行 3 次（5-2）。

  **i. 有多少次迭代没有分歧？**

  3 次迭代没有分歧（所有 1024 个线程都会执行）。

  **ii. 有多少次迭代存在分歧？**

  第 4 次和第 5 次迭代存在线程分歧。


### 习题 2
**对于向量加法，假设向量长度为 2000，每个线程计算一个输出元素，线程块大小为 512 个线程。网格中共有多少个线程？**

覆盖 2000 个元素所需的最少 512 大小的块数为 4，共 2048 个线程。

### 习题 3
**对于上一题，你预计有多少个 warp 会因为向量长度的边界检查而产生分歧？**

共有 `2048 / 32 = 64` 个 warp。覆盖线程 `2015-2047` 的 warp 将不活跃（被跳过）。覆盖线程 `1984-2015` 的 warp 将产生分歧：线程 1984-1999 有数据需要处理，而线程 `2000-2015` 没有。

### 习题 4
**考虑一个假设的块，其中有 8 个线程在到达屏障之前执行一段代码。各线程执行该段代码所需的时间（微秒）分别为：2.0、2.3、3.0、2.8、2.4、1.9、2.6 和 2.9；它们将剩余时间用于等待屏障。线程总执行时间中有多少百分比花在了等待屏障上？**

首先找到执行时间最长的线程，即 `3.0 微秒`。然后计算每个线程的等待时间之和：`(3.0 - 2.0) + (3.0 - 2.3) + (3.0 - 3.0) + (3.0 - 2.8) + (3.0 - 2.4) + (3.0 - 1.9) + (3.0 - 2.6) + (3.0 - 2.9) = 1.0 + 0.7 + 0.0 + 0.2 + 0.6 + 1.1 + 0.4 + 0.1 = 4.1`。总执行时间为 `8×3.0=24.0` 微秒。因此等待时间占比为 `4.1/24.0≈17%`。


### 习题 5
**一位 CUDA 程序员说，如果他们在每个块中只启动 32 个线程，就可以在需要屏障同步的地方省略 `__syncthreads()` 指令。你认为这是个好主意吗？请解释。**

由于所有线程都在同一个 warp 中处理，它们以 SIMD（单指令多数据）方式锁步执行。这意味着它们在指令级别上天然是同步的，因此理论上可以省略显式的 `__syncthreads()` 指令。然而在实践中，例如涉及内存操作时，这不一定成立。虽然指令会逐条执行，但底层硬件（如内存读写）可能由于各种硬件限制而产生延迟。因此在实践中，使用 `__syncthreads()` 更安全。此外，NVIDIA 不保证未来 warp 大小会保持在 32；为了构建面向未来的代码库，通常最好还是使用 `__syncthreads()`。


### 习题 6
**如果一个 CUDA 设备的 SM 最多支持 1536 个线程和最多 4 个线程块，以下哪种块配置能使 SM 中的线程数最多？**
   - **a. 每块 128 个线程**
   - **b. 每块 256 个线程**
   - **c. 每块 512 个线程**
   - **d. 每块 1024 个线程**

要确定哪种配置能产生最多线程，需要考虑两个限制：

- SM 最多支持 4 个块
- SM 最多支持 1536 个线程

对每种配置计算线程数：

- **a.** `min(4, 1536/128) × 128 = min(4, 12) × 128 = 4 × 128 = 512`
- **b.** `min(4, 1536/256) × 256 = min(4, 6) × 256 = 4 × 256 = 1024`
- **c.** `min(4, 1536/512) × 512 = min(4, 3) × 512 = 3 × 512 = 1536`
- **d.** `min(4, 1536/1024) × 1024 = min(4, 1) × 1024 = 1 × 1024 = 1024`

答案是 **c**。


### 习题 7
**假设一个设备每个 SM 最多允许 64 个块和 2048 个线程。判断以下每种 SM 分配方案是否可行，如果可行，指出占用率。**

**a. 8 个块，每块 128 个线程**
**b. 16 个块，每块 64 个线程**
**c. 32 个块，每块 32 个线程**
**d. 64 个块，每块 32 个线程**
**e. 32 个块，每块 64 个线程**

将块数乘以块大小，验证是否不超过 SM 支持的线程数。所有配置的块数都不超过 64 的限制。占用率 = 实际线程数 / SM 最大线程数（2048）。

- **a.** `8 × 128 = 1024`。`1024 ≤ 2048`，可行。占用率 `1024/2048 = 50%`
- **b.** `16 × 64 = 1024`。`1024 ≤ 2048`，可行。占用率 `1024/2048 = 50%`
- **c.** `32 × 32 = 1024`。`1024 ≤ 2048`，可行。占用率 `1024/2048 = 50%`
- **d.** `64 × 32 = 2048`。`2048 ≤ 2048`，可行。占用率 `2048/2048 = 100%`
- **e.** `32 × 64 = 2048`。`2048 ≤ 2048`，可行。占用率 `2048/2048 = 100%`

### 习题 8
**考虑一个具有以下硬件限制的 GPU：每个 SM 2048 个线程、每个 SM 32 个块、每个 SM 64K（65,536）个寄存器。对于以下每种内核特性，说明该内核能否达到完全占用。如果不能，指出限制因素。**

完全占用意味着 SM 能利用全部 2048 个线程。有两个限制因素：
1. SM 最多支持 32 个块
2. SM 最多支持 65,536 个寄存器

**a. 内核每块使用 128 个线程和每线程 30 个寄存器。**

基于块大小的最大块数：`2048 / 128 = 16`。16 低于硬件限制的 32 个块，满足约束。

每块寄存器数：`128 × 30 = 3,840`。基于寄存器的最大块数：`65,536 / 3,840 = 17`。17 大于块大小限制的 16。

因此该配置的线程总数为：`16 × 128 = 2048` 个线程，达到完全占用（`2048 / 2048 = 100%`）。

**b. 内核每块使用 32 个线程和每线程 29 个寄存器。**

基于块大小的最大块数：`2048 / 32 = 64`。64 超过了硬件限制的 32 个块，不满足约束。

每块寄存器数：`32 × 29 = 928`。基于寄存器的最大块数：`65,536 / 928 = 70`。同样超过 SM 的 32 块限制。

因此该配置的线程总数为 `32 × 32 = 1024` 个线程，占用率 `1024/2048 = 50%`。

限制因素是 SM 支持的最大块数。

**c. 内核每块使用 256 个线程和每线程 34 个寄存器。**

基于块大小的最大块数：`2048 / 256 = 8`。8 低于硬件限制的 32 个块，满足约束。

每块寄存器数：`256 × 34 = 8,704`。基于寄存器的最大块数：`65,536 / 8,704 = 7`。因此 7 成为新的块数上限。

线程总数为 `256 × 7 = 1,792` 个线程。占用率 `1,792 / 2048 = 87%`，未达到完全占用。

限制因素是寄存器数量。

### 习题 9
**一位学生提到他们使用 32×32 线程块的矩阵乘法内核成功地将两个 1024×1024 的矩阵相乘。该学生使用的 CUDA 设备每块最多允许 512 个线程，每个 SM 最多 8 个块。该学生还提到每个线程块中的每个线程计算结果矩阵的一个元素。你的反应是什么？为什么？**

结果矩阵有 `1024 × 1024 = 1,048,576` 个元素。学生使用的网格有 `32 × 32 = 1024` 个块，每块支持 `512` 个线程。网格中共有 `1024 × 512 = 524,288` 个线程。学生声称每个线程只计算结果矩阵的一个元素，但基于上述配置这是不可能的，线程数不够。每个 SM 的块数限制与此问题无关。
